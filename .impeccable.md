## Design Context

### Users
BeatLens serves two audiences equally: developers and tech enthusiasts exploring audio fingerprinting technology (who value accuracy and precision), and music lovers who just want to identify songs (who value simplicity and delight). The UI must feel approachable to casual users without dumbing itself down for technical ones. Every interaction should feel confident and instant.

### Brand Personality
**Sleek · Minimal · Premium**

The tone is quiet confidence — like a precision instrument that happens to be beautiful. No clutter, no noise. Every element earns its place. The interface should feel like something you'd find in a Linear dashboard or a Vercel deploy screen: dark, typographically clean, with subtle depth cues rather than loud visuals.

### Aesthetic Direction
- **Reference:** Linear and Vercel — ultra-minimal dark UI, precise developer aesthetic, restrained use of color, strong typography hierarchy
- **Anti-reference:** avoid music-app maximalism (Spotify's rich imagery, Shazam's neon energy), avoid gradients used gratuitously, avoid anything that feels "designed by a developer who added too many effects"
- **Theme:** Dark-only. The existing `gray-950` base is correct. Cyan/teal accent stays but should be used sparingly — reserved for primary actions and active states only, not decorative use.
- **Motion:** Subtle and purposeful. Animations should communicate state changes (recording, matching, result arriving), not add flair for its own sake.

### Design Principles

1. **Restraint over decoration** — Add elements only when they carry information or guide action. When in doubt, remove rather than add.

2. **State clarity above all** — The app has three distinct states (idle, recording, matching) and two outcome states (match found, no match). Each must be visually unambiguous at a glance, even without reading text.

3. **Typography does the heavy lifting** — Use font weight, size, and spacing to create hierarchy instead of color or decoration. The UI should look intentional even without any color applied.

4. **Accent color is precious** — Cyan/teal is the single accent. Use it only on primary interactive elements (the record button, active tab indicator, focus rings). Everywhere else: white, grays, and transparency.

5. **Density that respects both audiences** — Enough information density for developers (fingerprint count, confidence scores, offset), but the primary result (song title + confidence) should dominate visually so casual users get what they need instantly.
